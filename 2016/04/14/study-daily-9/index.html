js的闭包

闭包：内部作用域（相当于子）可以访问外部作用域（相当于父）的变量（以引用的方式调用，不会引起gc），而函数是js中唯一拥有自己作用域的结构，所以闭包依赖函数。利用闭包我们可以在函数外部访问到函数内部的变量（模拟私有方法）。

js的异步编程

异步是指执行顺序上不是一个一个任务来的（异步时不必要等待上一个任务完全执行），即函数并不立即返回结果，函数会把控制权交给回调，回调会在一段时间片内被调度，减少用户等待时间，提高用户体验。下面介绍js中几种异步编程方式：
1.利用回调（注意回调并不一定是异步的）

缺点：高耦合，难维护
优点:易于理解
2.事件监听

缺点：代码全变成事件驱动，使得流程不清晰
优点：易于模块化，低耦合
3.发布/订阅（观察者模式）
publish-subscribe,多种实现，利用信号量

4.promise

Javascript线程模型和事件驱动

Javascript在浏览器中的事件驱动（event driven）机制。事件驱动一般通过事件循环（event loop）和事件队列（event queue）来实现的。假定浏览器中有一个专门用于事件调度的实例（该实例可以是一个线程，我们可以称之为事件分发线程event dispatch thread），该实例的工作就是一个不结束的循环，从事件队列中取出事件，处理所有很事件关联的回调函数（event handler）。注意回调函数是在Javascript的主线程中运行的，而非事件分发线程中，以保证事件处理不会发生阻塞。
while(true) {
var event = eventQueue.pop();
if(event && event.handler) {
event.handler.execute(); // execute the callback in Javascript thread
} else {
sleep(); //sleep some time to release the CPU do other stuff
}
}
通过事件驱动机制，我们可以想象Javascript的编程模型就是响应一系列的事件，执行对应的回调函数。

js异步编程

异步编程一般应用于io（读写资源，牵涉到大量字节的处理，需要长时间等待）的操作，而所谓异步编程就是把一个任务分成两阶段，第一阶段发出一个请求，第二阶段等待响应（回调函数，promise的then()，jquery的on事件处理。。。都属于这个阶段的实现,不接收响应不会执行，自然不会阻塞之后的操作）。这两部分不是连续执行的，当第一部分执行完成后，函数（第一个任务）会交出执行权，这个时候之后的函数（第二个任务）获得执行权，开始执行，等第一个任务的第二阶段得到响应时，第二个任务暂停执行，又把执行权交还给第一个任务的第二阶段，此时，第二阶段在这段时间片内执行，从代码顺序上来看，异步执行是不连续的(a（1）->b()->a(2))，而同步则是严格的按照代码顺序执行（a(1,2)->b()），但是会阻塞。

callee与caller的区别简单介绍

1.callee属性
此属性属于arguments对象,返回正在执行的函数，通过arguments.callee实现了递归效果，如
function func(num){
if (num<=1) {
return 1;
}
else{
return num*arguments.callee(num-1);
};
};
console.log(func(5));
2.caller属性
此属性属于一个函数对象，它指向当前函数所属于的直接父函数，也可以说调用当前函数的函数。如
function CallLevel(){
if (CallLevel.caller == null)
return(“CallLevel was called from the top level.”);
else
return(“CallLevel was called by another function.”);
}

document.write(CallLevel());

// Output: CallLevel was called from the top level.

proto and prototype

proto是对象上的一个内置属性，他指向构造器的原型对象，prototype是构造器的原型对象（本质是一个指针），原型对象上的属性和方法可以被该原型的所有实例所共享。简而言之，obj.proto == objConstructor.prototype
构造器有5种，object，function，string，date，arrary
object上的静态方法：object.create(protoobj)可以传入一个原型对象，创造一个原型链干净的对象
object.defineProperty(o,prop,descriptor)直接定义对象上的属性，descriptor包含value，writable：是否可写，configurable：是否可删，enumerable：是否可以被枚举（for in ，object.keys()），set(属性访问器，set:function(val){prop= val},可带一个参数)，get（属性访问器，get:function(){dosomething();return prop},回调不带参数）
object.getOwnPropertyDescriptor(obj,prop)返回属性描述符desctriptor
object.keys(o),返回数组，对象的属性

递归

递归顾名思义就是先递后归，函数先一层一层地向下执行，直到遇到递归基（递归终止条件），开始从最后结果一层一层地回归到顶层。递归的时候每次调用会新开辟一片栈区，函数内部的局部变量会重新初始化，先要在递归中保持变量（引用），可以使用静态变量。在一个函数内部声明一个局部变量就相当于，把这个局部变量从栈区拉到全局区，调用时是对这个变量直接引用，而不是再重新开辟栈区，调用完后再销毁。
