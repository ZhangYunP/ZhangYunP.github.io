typeof 和 instanceof

typeof是一元运算符，后面接一个运算数，可以用来判断一个变量的类型，可能的结果有：number，boolean，object，function，string，undefined。也可以用它来判断一个变量是否存在，如if(typeof a != ‘undefined’){dosomething}，不要使用if(a){dosomething},这样如果a不存在就会报错。对于arrary，null它一律返回object，这也是他的局限性。
instanceof ：实例,例子
instanceof 用于判断一个变量是否某个对象的实例。如 var a=new Array();alert(a instanceof Array); 会返回 true，同时 alert(a instanceof Object) 也会返回 true;这是因为 Array 是 object 的子类。另外插一句，arguments不是arrary的实例。

defineProperty

Object.defineProperty(obj, prop, descriptor)方法接收三个参数：需要添加或修改属性的对象，属性名称，属性描述options

Object.defineProperty() 方法设置属性时，属性不能同时声明访问器属性（ set 和 get ）和 writable 或者 value 属性。 意思就是，某个属性设置了 writable 或者 value 属性，那么这个属性就不能声明 get 和 set 了，反之亦然。

JS的作用域

作用域分为全局和局部作用域，在es6之前，js没有块级作用域，但他有一个函数级作用域，在函数级作用域内部声明的变量可以看成局部变量，这些局部变量不能在外部被访问（除闭包外）。
函数级作用域

没有块级作用域

在函数内部没用关键词var声明的变量属于全局变量，全局变量优先级小于局部变量，当访问函数内的 foo 变量时，JavaScript 会按照下面顺序查找：

1.当前作用域内是否有 var foo 的定义。
2.函数形式参数是否有使用 foo 名称的。
3.函数自身是否叫做 foo。
4.回溯到上一级作用域，然后从 #1 重新开始。
全局变量属于window对象，可以表示为window.obj或’obj’ in window。另外，为了避免对全局作用域的污染，所以一般情况下我们尽可能少的声明全局变量。利用立即执行函数（匿名包装器）创建一个命名空间

变量提升

从代码顺序上看，js中我们可以先使用变量在声明，在执行顺序上看，在代码运行前，var 表达式和 function 声明都将会被提升到当前作用域的顶部。。

函数表达式也存在变量提升（当作一个特殊变量），函数声明会覆盖变量声明
